## Overview

This endpoint predicts whether a customer's credit risk is "good" or "bad" based on their demographic and financial information. The prediction is made using a PyTorch deep learning model that has been trained on historical credit data.

## Endpoint

```
POST /credit_score_prediction
```

## Request Schema

The request body must be a JSON object with the following fields:

### Required Fields

<ParamField body="Age" type="integer" required>
  Age of the applicant in years. Must be greater than 0.
  
  **Example**: `35`
</ParamField>

<ParamField body="Sex" type="string" required>
  Gender of the applicant.
  
  **Allowed values**: 
  - `"male"`
  - `"female"`
  
  **Example**: `"male"`
</ParamField>

<ParamField body="Job" type="string" required>
  Skill level of the applicant's job.
  
  **Allowed values**:
  - `"unskilled and non-resident"`
  - `"unskilled and resident"`
  - `"skilled"`
  - `"highly skilled"`
  
  **Example**: `"skilled"`
</ParamField>

<ParamField body="Housing" type="string" required>
  Type of housing for the applicant.
  
  **Allowed values**:
  - `"own"` - Owns housing
  - `"rent"` - Rents housing
  - `"free"` - Free housing
  
  **Example**: `"own"`
</ParamField>

<ParamField body="Saving accounts" type="string" required>
  Status of the savings account. Note the space in the field name.
  
  **Allowed values**:
  - `"NA"` - No savings account
  - `"little"` - Little savings
  - `"moderate"` - Moderate savings
  - `"quite rich"` - Quite rich savings
  - `"rich"` - Rich savings
  
  **Example**: `"NA"`
</ParamField>

<ParamField body="Checking account" type="string" required>
  Status of the checking account. Note the space in the field name.
  
  **Allowed values**:
  - `"NA"` - No checking account
  - `"little"` - Little balance
  - `"moderate"` - Moderate balance
  - `"rich"` - Rich balance
  
  **Example**: `"little"`
</ParamField>

<ParamField body="Credit amount" type="number" required>
  Amount of credit requested. Must be greater than 0.
  
  **Example**: `9055.0`
</ParamField>

<ParamField body="Duration" type="integer" required>
  Duration of the credit in months. Must be greater than 0.
  
  **Example**: `36`
</ParamField>

<ParamField body="Purpose" type="string" required>
  Purpose of the credit.
  
  **Allowed values**:
  - `"car"`
  - `"furniture/equipment"`
  - `"radio/TV"`
  - `"domestic appliances"`
  - `"repairs"`
  - `"education"`
  - `"business"`
  - `"vacation/others"`
  
  **Example**: `"education"`
</ParamField>

## Response Schema

The response is a JSON object with the following fields:

<ResponseField name="prediction" type="string" required>
  The predicted credit risk category.
  
  **Possible values**:
  - `"good"` - Good credit risk
  - `"bad"` - Bad credit risk
</ResponseField>

<ResponseField name="probability" type="number" required>
  The probability that the credit risk is "good". Value between 0 and 1.
  
  **Range**: `0.0` to `1.0`
  
  **Note**: Higher values indicate higher confidence in a "good" prediction.
</ResponseField>

## Inference Pipeline

The prediction process follows these steps:

<Steps>
  <Step title="Input Validation">
    The request data is validated against the `CreditRiskInput` Pydantic schema to ensure all fields are present and have correct types and values.
  </Step>
  
  <Step title="Data Preprocessing">
    The input is transformed into a pandas DataFrame and processed using a scikit-learn preprocessor that handles:
    - Categorical encoding
    - Feature scaling
    - Feature engineering
  </Step>
  
  <Step title="Model Inference">
    The preprocessed data is converted to a PyTorch tensor and passed through the deep learning model:
    - The model outputs probabilities for both "bad" and "good" classes
    - The prediction is determined by the class with the highest probability
  </Step>
  
  <Step title="Response Generation">
    The prediction result and probability are packaged into a `CreditRiskOutput` response object and returned as JSON.
  </Step>
</Steps>

## Important Notes

<Warning>
  **Field Names with Spaces**: The fields `"Saving accounts"` and `"Checking account"` contain spaces. Ensure you include the space when making requests, or the validation will fail.
</Warning>

<Info>
  **Singleton Pattern**: The model is loaded once at startup using a singleton pattern. All prediction requests share the same model instance for optimal performance.
</Info>

<Note>
  **CPU Inference**: The model runs on CPU by default. For high-throughput production deployments, consider configuring GPU support.
</Note>
