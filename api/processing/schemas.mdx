This module defines the input and output schemas for the credit risk prediction API using Pydantic for data validation.

## Input Schemas

### CreditRiskInput

Defines the structure of input data for credit risk prediction. Field names match the original dataset columns.

```python
class CreditRiskInput(BaseModel)
```

<ParamField path="Age" type="int" required>
  Age of the applicant in years. Must be greater than 0.
</ParamField>

<ParamField path="Sex" type="SexEnum" required>
  Sex of the applicant. Valid values: `male`, `female`
</ParamField>

<ParamField path="Job" type="JobEnum" required>
  Job skill level of the applicant. See [JobEnum](#jobenum) for valid values.
</ParamField>

<ParamField path="Housing" type="HousingEnum" required>
  Housing type of the applicant. Valid values: `own`, `rent`, `free`
</ParamField>

<ParamField path="Saving accounts" type="SavingAccountsEnum" required>
  Status of the applicant's savings account. See [SavingAccountsEnum](#savingaccountsenum) for valid values.
</ParamField>

<ParamField path="Checking account" type="CheckingAccountEnum" required>
  Status of the applicant's checking account. See [CheckingAccountEnum](#checkingaccountenum) for valid values.
</ParamField>

<ParamField path="Credit amount" type="float" required>
  Amount of credit requested. Must be greater than 0.
</ParamField>

<ParamField path="Duration" type="int" required>
  Duration of the credit in months. Must be greater than 0.
</ParamField>

<ParamField path="Purpose" type="PurposeEnum" required>
  Purpose of the credit. See [PurposeEnum](#purposeenum) for valid values.
</ParamField>

<Note>
  The schema uses field aliases to support spaces in field names (e.g., "Saving accounts", "Checking account", "Credit amount"). Both the aliased and underscore versions are accepted.
</Note>

<CodeGroup>
```json Example Request
{
  "Age": 35,
  "Sex": "male",
  "Job": "unskilled and resident",
  "Housing": "free",
  "Saving accounts": "NA",
  "Checking account": "NA",
  "Credit amount": 9055,
  "Duration": 36,
  "Purpose": "education"
}
```

```python Python Example
from server.schemas import CreditRiskInput

input_data = CreditRiskInput(
    Age=35,
    Sex="male",
    Job="unskilled and resident",
    Housing="free",
    Saving_accounts="NA",  # Can use underscore version
    Checking_account="NA",
    Credit_amount=9055,
    Duration=36,
    Purpose="education"
)
```
</CodeGroup>

## Validation

All schemas include Pydantic validation:

- **Type validation**: Ensures all fields match their declared types
- **Range validation**: Age, Credit amount, and Duration must be greater than 0
- **Enum validation**: Categorical fields must use valid enum values
- **Probability validation**: Output probability must be between 0.0 and 1.0

<CodeGroup>
```python Validation Example
from server.schemas import CreditRiskInput
from pydantic import ValidationError

try:
    # This will raise a validation error (Age must be > 0)
    invalid_input = CreditRiskInput(
        Age=-5,
        Sex="male",
        Job="skilled",
        Housing="own",
        Saving_accounts="little",
        Checking_account="moderate",
        Credit_amount=5000,
        Duration=24,
        Purpose="car"
    )
except ValidationError as e:
    print(e)
```
</CodeGroup>
